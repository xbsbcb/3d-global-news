# 3D全球实时新闻数据整合 - 项目计划书

> 版本：v1.0
> 创建日期：2026-03-26
> 项目类型：Web端3D数据可视化 / MVP原型

---

## 一、项目概述

### 1.1 项目名称
**GlobeNews** - 3D全球新闻可视化平台

### 1.2 核心价值
在网页端通过3D地球可视化展示全球实时新闻数据，用户可以直观地浏览、搜索和探索来自世界各地的新闻事件，按地理区域和分类进行信息整合。

### 1.3 目标用户
- 新闻从业者：快速了解全球动态
- 研究人员：舆情分析和数据研究
- 普通用户：了解国际时事
- 学习者：3D可视化技术实践

---

## 二、技术选型

### 2.1 前端技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | React 18 | 组件化开发，生态成熟 |
| 3D渲染 | Three.js + Globe.gl | 3D地球可视化核心 |
| 状态管理 | Zustand | 轻量级状态管理 |
| 样式 | Tailwind CSS | 原子化CSS，快速开发 |
| 构建工具 | Vite | 快速开发服务器和构建 |

### 2.2 后端技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| 语言 | Node.js / TypeScript | 类型安全 |
| 框架 | Express.js | 轻量REST API |
| 数据库 | PostgreSQL | 结构化数据存储 |
| RSS处理 | RSSHub + 自建解析器 | 新闻数据采集 |

### 2.3 部署方案
- 容器化：Docker
- Web服务器：Nginx
- 部署模式：内网/本地部署

## 三、功能需求

### 3.1 核心功能（优先级排序）

#### P0 - MVP必须功能
1. **3D地球渲染**
   - Globe.gl实现的3D地球
   - 暗色NASA风格主题
   - 洲际区域划分显示
   - 缩放、旋转、拖拽交互

2. **新闻数据展示**
   - 新闻标记点显示（按地理坐标）
   - 点击标记查看新闻摘要
   - 新闻列表展示

3. **数据采集**
   - RSSHub + GitHub新闻源
   - 每日汇总更新（可配置间隔）
   - 后端定时任务抓取

4. **搜索与过滤**
   - 关键词搜索
   - 分类过滤（金融、政治、财经、科技）

#### P1 - 第二阶段功能
5. **地区特写**
   - 点击区域放大到洲际特写
   - 局部地区新闻聚合展示

6. **时间线浏览**
   - 按时间轴查看历史新闻
   - 时间滑块控制

7. **热力图模式**
   - 新闻密度热力图展示
   - 可切换点标记/热力图

#### P2 - 规划中功能
8. **弧线连接动画**
   - 新闻传播路径可视化

9. **多语言翻译**
   - 英文新闻自动翻译为中文

### 3.2 用户交互流程
```
用户打开页面 → 3D地球加载 → 新闻点标记展示
    ↓
用户操作地球（旋转/缩放）→ 查看特定区域新闻
    ↓
点击新闻点 → 弹出新闻详情卡片
    ↓
使用搜索/分类 → 过滤新闻
    ↓
时间线滑动 → 浏览历史新闻
```

## 四、数据规格

### 4.1 新闻数据模型
```typescript
interface NewsItem {
  id: string;              // 唯一标识
  title: string;           // 标题
  summary: string;        // 摘要
  content: string;        // 完整内容
  source: string;         // 来源
  sourceUrl: string;      // 原文链接
  category: string;       // 分类
  country: string;        // 国家
  city: string;           // 城市
  latitude: number;        // 纬度
  longitude: number;       // 经度
  publishedAt: Date;      // 发布时间
  fetchedAt: Date;        // 抓取时间
  language: string;       // 语言
}
```

### 4.2 数据源规划
| 来源 | 类型 | 说明 |
|------|------|------|
| RSSHub | RSS聚合 | 聚合多源RSS |
| GitHub Trending | 开发者新闻 | 技术类新闻补充 |
| 备用源 | 公开API | NewsAPI等作为补充 |

### 4.3 存储规划
- **PostgreSQL**：新闻内容、用户配置
- **缓存**：热点新闻Redis缓存（可选）

## 五、UI/UX设计方向

### 5.1 视觉风格
- **主题**：暗色地理风格（NASA风格）
- **背景**：深空黑色 (#0a0a0f)
- **地球**：暗色调底图，发光边界
- **标记点**：荧光色点，大小表示新闻数量
- **字体**：Inter / Noto Sans SC（中文）

### 5.2 配色方案
```
主背景：#0a0a0f (深空黑)
次背景：#1a1a2e (深紫黑)
主色调：#00d4ff (科技蓝)
强调色：#ff6b6b (警示红)
成功色：#4ecdc4 (青绿)
文字色：#ffffff / #a0a0a0
```

### 5.3 布局结构
```
┌─────────────────────────────────────┐
│ Header (Logo + 搜索 + 分类筛选)      │
├─────────────────────────────────────┤
│                                     │
│         3D Globe (主视觉区)          │
│                                     │
├─────────────────────────────────────┤
│ Timeline (时间线)                   │
├─────────────────────────────────────┤
│ Footer (数据源 + 更新状态)           │
└─────────────────────────────────────┘
```

### 5.4 响应式策略
- **桌面端 (>1024px)**：全功能展示
- **平板端 (768-1024px)**：简化侧边栏
- **移动端 (<768px)**：全屏地图 + 底部抽屉

## 六、性能要求

### 6.1 渲染性能
- 目标帧率：60fps
- 标记点数量支持：>1000个
- 缩放/旋转响应：<16ms

### 6.2 加载性能
- 首屏加载：<5秒（可接受加载动画）
- 地图资源：懒加载 + CDN加速

### 6.3 数据更新
- 更新间隔：默认每日（用户可配置）
- 后台抓取，不阻塞前端

## 七、开发计划

### 7.1 项目周期（1-2周 MVP）

| 阶段 | 时间 | 任务 |
|------|------|------|
| Day 1 | 环境搭建 | 项目初始化、Docker配置 |

| Day 8-9 | 优化完善 | 性能优化、样式调整 |
| Day 10-14 | 测试部署 | Bug修复、本地部署 |

### 7.2 里程碑
- **M1 (Day 5)**：3D地球可展示新闻
- **M2 (Day 9)**：完成搜索和分类功能
- **M3 (Day 14)**：MVP完成，可本地运行

## 八、风险评估

### 8.1 已识别风险
| 风险 | 等级 | 应对措施 |
|------|------|----------|
| RSS源不稳定 | 低 | 冗余数据源，容错处理 |
| 3D性能问题 | 中 | 渐进加载，点聚合 |
| 数据解析失败 | 低 | 格式校验，降级策略 |

### 8.2 内网部署限制
- 无外网访问需求
- RSSHub可内网自建
- 完全离线可用
