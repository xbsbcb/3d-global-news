# GlobeNews - 3D全球新闻可视化平台

> 版本：v2.0
> 创建日期：2026-03-31
> 项目类型：Web端3D数据可视化 / MVP原型

---

## 一、项目概述

### 1.1 项目名称
**GlobeNews** - 3D全球新闻可视化平台

### 1.2 核心价值
通过3D粒子地球可视化展示全球实时新闻数据，用户可以直观地浏览、搜索和探索来自世界各地的新闻事件，按地理区域和分类进行信息整合。

### 1.3 目标用户
- 普通大众：关心国际时事的普通用户
- 新闻爱好者：希望直观了解全球新闻分布

### 1.4 核心约束
- **本地优先**：所有功能尽可能本地实现，减少外部依赖
- **免费优先**：优先使用免费数据源和工具

## 二、技术架构

### 2.1 前端技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Vue 3 | 组件化开发，Composition API |
| 3D渲染 | Three.js | WebGL核心，支持粒子系统 |
| 动画 | GSAP | 高性能动画引擎 |
| 地理数据 | D3-geo | 经纬度转3D坐标 |
| 粒子特效 | GLSL Shader | GPU高性能粒子渲染 |
| 构建工具 | Vite | 快速开发服务器和构建 |

**前端模块架构**（参考 `docx/前端.md`）：
- `EarthScene`：主场景控制（渲染器、相机、灯光）
- `ParticleEarth`：粒子地球与聚拢特效
- `GeoLayer`：地理边界与区域显示
- `DataLayer`：热力图、连接弧线
- `InteractionManager`：局部特写、缩放交互

### 2.2 后端技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 语言 | Rust | 高性能、安全 |
| Web框架 | Axum | 轻量异步框架 |
| 数据库 | SQLite | 轻量文件数据库 |
| ORM | Diesel / SeaORM | Rust ORM |
| API文档 | Swagger/OpenAPI | 自动生成 |

### 2.3 数据源

| 来源 | 类型 | 说明 |
|------|------|------|
| worldnewsapi.com | REST API | 全球新闻 |
| gnews.io | REST API | 多语言新闻 |
| newsdata.io | REST API | 新闻数据 |


## 三、功能需求

### 3.1 核心功能（v1.0 MVP）

| 优先级 | 功能 | 描述 |
|--------|------|------|
| P0 | 3D粒子地球 | 粒子构成主体，缩放聚拢特效 |
| P0 | 地球交互 | 旋转、缩放、拖拽 |
| P0 | 新闻标记 | 按坐标显示新闻点 |
| P0 | 新闻详情 | 点击查看摘要 |
| P0 | 数据采集 | 定时从API抓取新闻 |
| P0 | 搜索功能 | 关键词全文搜索 |
| P1 | 分类筛选 | 固定+动态混合分类 |
| P1 | 地区特写 | 点击区域放大 |
| P2 | 热力图 | 新闻密度热力图 |
| P2 | 连接线 | 新闻传播路径弧线 |

### 3.2 用户交互流程

```
用户打开页面 → 3D粒子地球加载 → 新闻点标记展示
    ↓
用户操作地球（旋转/缩放）→ 查看特定区域新闻
    ↓
点击新闻点 → 弹出新闻详情卡片
    ↓
使用搜索/分类 → 过滤新闻
    ↓
点击区域 → 局部特写放大
```

### 3.3 新闻分类方案

**固定分类**：政治、财经、科技、社会、文化、娱乐、体育

**动态分类**：根据新闻关键词自动打标签

## 四、数据规格

### 4.1 新闻数据模型

```rust
struct News {
    id: i64,
    title: String,           // 标题
    summary: String,         // 摘要
    content: String,         // 完整内容
    source: String,         // 来源
    source_url: String,     // 原文链接
    category: String,       // 分类
    country: String,        // 国家
    city: String,           // 城市
    latitude: f64,          // 纬度
    longitude: f64,         // 经度
    published_at: DateTime, // 发布时间
    fetched_at: DateTime,  // 抓取时间
}
```

### 4.2 地理坐标策略

根据实际API返回内容决定：
- 优先使用API返回的地理坐标
- 备选：从新闻内容提取地名，映射到坐标
- 最终备选：使用国家/地区中心坐标
